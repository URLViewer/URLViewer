# デプロイ方法と仕組み

最終更新: 2026-05-03

## 1. デプロイ対象
- 対象リポジトリ: `URLViewer/UrlViewer-App`
- 配布形式: Windows（NSIS）
- ビルドツール: `electron-builder`

## 2. ローカル配布ビルド

```bash
yarn install
yarn dist
```

- `dist` は `yarn build` + `electron-builder` を実行。
- 出力先は `UrlViewer-App/release/`。

## 3. 自動リリース（GitHub Actions）

### 3.1 発火条件
- `push tags: v*`
- `workflow_dispatch`（手動実行）

### 3.2 タグ実行時の流れ
1. Checkout
2. Node.js 24 セットアップ
3. Corepack 有効化
4. `yarn install --immutable`
5. `RELEASE_PAT` 存在チェック
6. `GH_TOKEN=${{ secrets.RELEASE_PAT }}` で `yarn dist:publish` 実行

### 3.3 公開コマンド
`dist:publish` は次を実行する。

```bash
yarn build && electron-builder --publish onTag
```

- `--publish onTag` によりタグ時のみ公開。
- 誤実行時の GitHub Release 422 を回避。

### 3.4 手動実行（workflow_dispatch）
- `yarn dist` の build-only 実行。
- Release publish は行わない。

## 4. electron-builder publish 設定
`package.json` の `build.publish`:

- provider: `github`
- owner: `URLViewer`
- repo: `UrlViewer-App`
- releaseType: `release`

## 5. 自動アップデート連携の仕組み

### 5.1 実行条件
- `app.isPackaged === true` で既定有効。
- 開発時は既定無効（`URLVIEWER_ENABLE_DEV_UPDATER=1` で有効化可能）。

### 5.2 起動時フロー
1. `checkForUpdates()` 実行
2. 更新があれば自動ダウンロード
3. ダウンロード完了で再起動確認ダイアログ表示
4. 承認時 `quitAndInstall()`

### 5.3 制御用環境変数
- `URLVIEWER_DISABLE_UPDATER=1`: updater 無効化
- `URLVIEWER_ENABLE_DEV_UPDATER=1`: 開発時にも有効化
- `URLVIEWER_ALLOW_PRERELEASE_UPDATES=1`: prerelease も更新対象

## 6. 運用チェックポイント
- タグは必ず `v` プレフィックスを使用（例: `v1.2.3`）。
- `RELEASE_PAT` 未設定時は workflow が明示的に失敗する。
- `package.json` の version とタグ整合を保つ。
