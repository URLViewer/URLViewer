# トークン取得方法と利用箇所

最終更新: 2026-05-03

## 1. このプロジェクトで使うトークンの種類

### 1.1 `RELEASE_PAT`（GitHub Actions用）
- 用途: GitHub Releases への publish
- 設定場所: `URLViewer/UrlViewer-App` リポジトリの Actions Secret
- secret 名: `RELEASE_PAT`
- workflow では `GH_TOKEN` 環境変数として `electron-builder` に渡される

### 1.2 private plugin 用 token（アプリ入力用）
- 用途: private Git リポジトリの clone/pull 認証
- 入力場所: アプリのプラグイン導入画面 `Token (private用)`
- 保存場所: OS キーチェーン（`keytar`）
  - service: `m3u8-viewer-plugins`
  - account: `pluginId`

## 2. 取得方法

## 2.1 `RELEASE_PAT` の取得
1. GitHub の token 作成画面で PAT を新規作成（fine-grained 推奨）。
2. 対象リポジトリを `URLViewer/UrlViewer-App` に限定。
3. Release 作成・アセットアップロードができる権限を付与。
4. 発行した token をコピー。
5. `Settings > Secrets and variables > Actions > New repository secret` で `RELEASE_PAT` を登録。

## 2.2 private plugin token の取得
1. GitHub の token 作成画面で PAT を新規作成。
2. 対象を private plugin リポジトリに限定。
3. clone/pull に必要な読み取り権限を付与。
4. 発行した token をアプリの `Token (private用)` 欄へ入力して導入。

## 3. なぜ token が必要か

### 3.1 リリース用
- GitHub Actions の既定トークン権限が制限される環境でも publish を成功させるため。
- workflow は `RELEASE_PAT` 未設定時に失敗し、設定漏れを検出する。

### 3.2 private plugin 用
- private リポジトリは匿名 clone/pull できないため。
- `installFromGit` / `update` 実行時に `isomorphic-git` の `onAuth` で使用する。

## 4. どこで使われるか（コード上）

### 4.1 `RELEASE_PAT`
- `UrlViewer-App/.github/workflows/release.yml`
  - `GH_TOKEN: ${{ secrets.RELEASE_PAT }}`
  - `yarn dist:publish` 実行時に利用

### 4.2 private plugin token
- `UrlViewer-App/src/electron/services/pluginManager.ts`
  - install: `git.clone(... onAuth ... payload.token)`
  - update: `git.pull(... onAuth ... savedToken)`
- `UrlViewer-App/src/electron/services/keychain.ts`
  - `savePluginToken` / `getPluginToken` / `deletePluginToken`

## 5. セキュリティ運用ルール
- token はコード・ドキュメント・コミットに含めない。
- release token と plugin token を使い回さない。
- 用途ごとに最小権限・短い有効期限を設定する。
- 不要になった token は失効し、必要に応じて再発行する。
