# アプリ仕様書

最終更新: 2026-05-03（現行コード基準）

## 1. 概要
`UrlViewer-App` は、URLを動画ソースとして登録・管理し、複数タブで再生する Electron アプリ。

- フレームワーク: Electron + React + TypeScript + Vite
- 主要機能: URL登録/検証、ライブラリ管理、グループ管理、タブ再生、プラグイン管理、自動アップデート

## 2. 実行アーキテクチャ

### 2.1 Main Process
- ウィンドウ生成、IPC ハンドラ登録、永続化、プラグイン導入、更新確認を担当。
- 起動時に `updateProvider.checkForUpdates()` を実行（条件付き）。

### 2.2 Preload
- `window.m3u8Viewer` API を公開し、Renderer から IPC を安全に呼び出す。

### 2.3 Renderer
- UI/状態管理（Zustand）を担当。
- 入力、ライブラリ、グループ、プラグイン、設定、キュー、プレイヤーを構成。

## 3. 機能仕様

### 3.1 URL入力と登録
- 入力は改行/空白/カンマ区切りで複数URLを受け付ける。
- URLは重複除去して処理。
- 検証モード:
  - `on-register`: 入力後すぐ検証・登録
  - `manual`: 検証待ちキューに蓄積し、手動実行

### 3.2 URL検証
- URL構文を検証後、到達性確認を行う。
- 到達性確認:
  1. `HEAD` リクエスト
  2. 405/501 の場合のみ `GET (Range: bytes=0-0)` へフォールバック
- タイムアウトは設定値（1000〜20000ms、下限1000ms）を適用。
- 検証並列数は 1〜8。

### 3.3 ライブラリ/タブ
- 登録済み動画一覧、名前変更、削除、全件削除。
- タブの開閉、アクティブ切替、起動時復元（設定でON/OFF）。
- ショートカット `Ctrl/Cmd + W` でアクティブタブを閉じる。

### 3.4 再生
- 再生は playback plugin 解決結果で実行。
- ビルトイン再生プラグイン:
  - `builtin.playback.hls`（`.m3u8` 優先）
  - `builtin.playback.html5`（m3u8以外）
- 再生位置は定期保存（resumeSeconds）。
- 再生エラー時:
  - `access-error`: 該当動画をライブラリから除外
  - `not-playable` / `unknown`: メッセージ表示

### 3.5 グループ
- グループ名は 1〜10 文字。
- 動画は複数グループに所属可能。
- プレイヤー上からもグループ追加可能。

### 3.6 プラグイン管理
- 一覧表示、詳細表示、ON/OFF、順序変更、更新（gitのみ）、削除。
- 導入経路: `zip` / `folder` / `git`。

### 3.7 エクスポート
- ライブラリ中のURLを重複除去し、`alive_urls_YYYYMMDD_HHMMSS.txt` 形式で保存。

## 4. 永続化データ

### 4.1 settings
- `restoreTabsOnLaunch: boolean`
- `validationMode: "on-register" | "manual"`
- `validationConcurrency: 1..8`
- `validationTimeoutMs: 1000..20000`

### 4.2 library
- `videos[]`: `id`, `label`, `sourceUrl`, `resumeSeconds?`, `lastValidatedAt?`, `addedByPluginId?`
- `groups[]`: `id`, `name`, `videoIds[]`
- `tabs`: `openVideoIds[]`, `activeVideoId`

### 4.3 plugins
- `schemaVersion: 1`
- `items[]`: `id`, `enabled`, `order`, `sourceType`, `sourceRef`, `localPath`, `manifest`, `lastUpdatedAt`

## 5. 既知の制約（重要）
- 外部 `input-panel` プラグインは main process で実行される。
- 外部 `playback` プラグインの動的実行は未対応（再生はビルトイン playback plugin で解決）。
