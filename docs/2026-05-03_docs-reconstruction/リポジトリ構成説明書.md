# リポジトリ構成説明書

最終更新: 2026-05-03

## 1. 全体構成
`UrlViewer` は meta リポジトリで、3つの submodule を束ねる。

- `UrlViewer-App`: Electron アプリ本体
- `UrlViewer-SDK`: プラグイン契約型 SDK
- `UrlViewer-Plugins`: 公開対象プラグインの配布ポリシー管理

`.gitmodules` で各 submodule URL を固定している。

## 2. ルート構成

```text
UrlViewer/
├─ UrlViewer-App/         # アプリ本体（submodule）
├─ UrlViewer-SDK/         # SDK（submodule）
├─ UrlViewer-Plugins/     # 公開プラグイン方針（submodule）
├─ docs/                  # ルート側の作業記録・統合ドキュメント
├─ README.md
└─ .gitmodules
```

## 3. UrlViewer-App の主要ディレクトリ

```text
UrlViewer-App/
├─ src/
│  ├─ electron/
│  │  ├─ main/            # Electron起動
│  │  ├─ ipc/             # IPCハンドラ
│  │  ├─ services/        # plugin管理・keychain・URL検証・updater
│  │  ├─ preload/         # contextBridge API
│  │  └─ store/           # electron-store永続化
│  ├─ web/
│  │  ├─ features/        # 画面機能（入力/ライブラリ/グループ/プラグイン等）
│  │  ├─ store/           # Zustand状態管理
│  │  └─ plugins/         # Renderer plugin runtime
│  └─ shared/             # 共通型・zod schema・既定値
├─ docs/                  # App側の実装履歴
├─ test/                  # unit/e2e
└─ .github/workflows/     # release workflow
```

## 4. UrlViewer-SDK の役割
- `PluginManifestV1`、`RendererPluginDefinition` など plugin 契約型を提供。
- App/Plugin 双方の型依存点を一元化。

## 5. UrlViewer-Plugins の役割
- 公開可能プラグインの運用ポリシーを管理。
- 各公開プラグインの個別リポジトリを submodule として束ねる。
- private plugin の実体・識別子は含めない。

## 6. 運用コマンド（ルート）

### clone
```bash
git clone --recurse-submodules https://github.com/URLViewer/UrlViewer.git
```

### submodule 同期
```bash
git submodule sync --recursive
git submodule update --init --recursive
```

## 7. セキュリティ境界
- private plugin 情報は公開リポジトリへ含めない。
- private plugin は別 private リポジトリで管理し、必要時にアプリUIから導入する。
