# プラグインシステム仕様と作成方法

最終更新: 2026-05-03（現行実装準拠）

## 1. プラグインシステム概要

### 1.1 契約型
`UrlViewer-SDK/src/index.ts` で plugin 契約を定義。

- `PluginManifestV1`
- `RendererInputPlugin`
- `RendererPlaybackPlugin`
- `RendererPluginDefinition`

### 1.2 capability
- `input-panel`
- `playback`

### 1.3 導入元
- `builtin`
- `zip`
- `folder`
- `git`

## 2. マニフェスト仕様（plugin.json）

必須:
- `id`
- `name`
- `version`
- `apiVersion`（現状固定: `"1.0.0"`）
- `entry`
- `capabilities`（`input-panel` / `playback` のいずれかを1つ以上）

任意:
- `description`（`string` または `{ summary, detailed? }`）
- `author`（`name`, `url?`）
- `homepage`
- `repository`
- `license`

例:
```json
{
  "id": "com.example.my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "apiVersion": "1.0.0",
  "entry": "./dist/index.js",
  "capabilities": ["input-panel"],
  "description": {
    "summary": "入力をURL候補へ変換",
    "detailed": "社内フォーマットをURLへ展開する"
  },
  "author": { "name": "Example Team" },
  "license": "MIT"
}
```

## 3. インストール時の判定
- `plugin.json` が読めること。
- schema 検証に通ること。
- 既存 plugin と `id` が重複しないこと（重複時は拒否）。

## 4. 現在の runtime 実行仕様（重要）
- `input-panel`:
  - 外部 plugin を main process 側で dynamic import して実行する。
  - `plugins:listPanels` で panel 情報を解決し、`plugins:resolveInput` で URL候補を実行取得する。
- `playback`:
  - Renderer runtime レジストリ（ビルトイン `hls/html5`）で解決する。
  - 外部 playback plugin の動的実行は未対応。

## 5. プラグイン作成手順

### 5.1 基本手順
1. `plugin.json` を作成し、必須フィールドを定義。
2. `entry` が指すビルド成果物（例: `dist/index.js`）を生成。
3. zip/folder/git のいずれかで導入可能な構造にする。

### 5.2 型定義の利用
```ts
import type { RendererPluginDefinition } from "@m3u8viewer/plugin-sdk";
```

### 5.3 推奨ディレクトリ構成
```text
my-plugin/
├─ plugin.json
├─ src/
│  └─ index.ts
├─ dist/
│  └─ index.js
└─ package.json
```

## 6. 実運用の注意
- private plugin は token 前提の git 導入で運用する。
- `id` は永続データ上のキーになるため変更しない。
- `apiVersion` は現行 App 実装と一致させる。
- 外部 playback plugin を使う場合は、App 側の playback runtime 拡張が別途必要。
